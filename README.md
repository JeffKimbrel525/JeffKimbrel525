# About Jeff

My interest in computers goes back further than I can remember, and my first real job turned that into a career — working full time in a computer repair shop in Wichita, Kansas as one of a team of six to eight technicians.

From there I moved into technical support roles, including business internet support where I regularly worked with DNS records, domain-hosted email, and server configuration — experience that would turn out to be directly relevant years later.

Most recently, while working for a residential lighting and fan retailer, I took on an infrastructure project that had nothing to do with my job title: rebuilding the store's network from a peer-to-peer setup into a proper domain environment running ESXi with Windows Server 2008 R2 Virtual Machines, Active Directory, and Remote Desktop Services. Before touching anything in production, I built the equivalent setup at home to test it first. That habit — standing something up at home before relying on it — stuck with me. When the company's second location needed the same upgrade, I led that one too, this time on Windows Server 2022.

In early 2026 I spun up that home server again, this time running Proxmox with a Pi-hole DNS server and a PostgreSQL instance I planned to use for practice during a data analytics bootcamp. When it came time to choose a capstone project, the answer was already running on my network: 21 IoT devices quietly making DNS requests around the clock with no one watching. The result was a full-pipeline analytics project — from automated log ingestion through statistical anomaly detection and interactive visualization — that found real, undisclosed device behavior hiding in plain sight.
